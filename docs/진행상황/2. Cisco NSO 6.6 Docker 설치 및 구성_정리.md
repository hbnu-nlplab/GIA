# Cisco NSO 6.6 Docker 설치 및 구성 가이드
![[IMG-20251122124941548.png]]
## 개요

이 문서는 Cisco NSO (Network Services Orchestrator) 6.6을 Docker 환경에서 설치하고 운영하는 방법에 대한 가이드입니다.

원활한 Pnetlab 장비관리를 위해서 Pnetlab이 설치된 PC와 admin PC를 별도로 구성하여 프로젝트를 진행했습니다.

**참고**: 로컬에서 빌드하지 않고 Docker Hub에서 미리 빌드된 이미지를 사용할 수 있습니다. (섹션 8. Docker Hub 이미지 활용 참고)

---

## 1. 개요 및 사전 요구사항

### 1.1 문서의 목적

본 가이드는 기존의 muhammad-rafi/cisco-nso-docker 저장소를 NSO 6.6 버전으로 업데이트하여 작성되었습니다. Docker를 활용하여 별도의 가상 머신 없이 NSO 환경을 빠르게 구축할 수 있도록 정리했습니다.

### 1.2 기술 스택

- **NSO 버전**: 6.6 (Free Trial)
- **기본 이미지**: Ubuntu 20.04 LTS
- **Java 버전**: OpenJDK 17
- **지원 NED**: Cisco IOS CLI 6.110.3, Cisco NX CLI 5.31
- **프로토콜 지원**: SSH CLI, NETCONF, RESTCONF

### 1.3 시스템 요구사항

| 항목 | 요구사항 | 설명 |
|------|--------|------|
| CPU | 2 cores 이상 | NSO Java VM 실행용 |
| 메모리 | 4GB 이상 | 권장: 8GB |
| 디스크 | 15GB 여유공간 | Docker 이미지 및 컨테이너용 |
| Docker | 20.10 이상 | BuildKit 지원 필수 |
| OS | Linux, macOS, Windows(WSL2) | Docker Desktop 또는 Docker CE |

## 4. Dockerfile 기술 사항

### 4.1 Dockerfile 개요

Dockerfile은 Docker 이미지를 구성하기 위한 명령어 집합입니다. 본 프로젝트의 Dockerfile은 다음 단계를 실행합니다:

1. Ubuntu 20.04 기본 이미지 선택
2. 필수 시스템 패키지 설치
3. NSO 설치 파일 복사 및 설치
4. NED 패키지 복사
5. 포트 노출 설정

### 4.2 주요 수정사항

원본 Dockerfile (NSO 5.5용)을 NSO 6.6으로 업데이트했습니다.

#### 4.2.1 MAINTAINER → LABEL 변경

MAINTAINER 명령어는 deprecated되었으며, LABEL을 사용하는 것이 모범 사례입니다.

**변경 전 (deprecated):**
```dockerfile
MAINTAINER Muhammad Rafi
```

**변경 후 (권장):**
```dockerfile
LABEL maintainer="Muhammad Rafi" \
      image.authors="murafi@cisco.com" \
      image.version="6.6"
```

#### 4.2.2 NSO 버전 및 경로 업데이트

모든 NSO 관련 경로를 5.5에서 6.6으로 변경합니다.

**변경 전 (NSO 5.5):**
```dockerfile
COPY nso-5.5.linux.x86_64.installer.bin .
RUN sh nso-5.5.linux.x86_64.installer.bin --local-install ~/nso-5.5
RUN echo "source $HOME/nso-5.5/ncsrc" >> ~/.bashrc
SHELL ["/bin/bash", "-c", "source $HOME/nso-5.5/ncsrc"]
COPY packages/ncs-5.5-cisco* /root/nso-5.5/packages/neds/
```

**변경 후 (NSO 6.6):**
```dockerfile
COPY nso-6.6.linux.x86_64.installer.bin .
RUN sh nso-6.6.linux.x86_64.installer.bin --local-install ~/nso-6.6
RUN echo "source $HOME/nso-6.6/ncsrc" >> ~/.bashrc
SHELL ["/bin/bash", "-c", "source $HOME/nso-6.6/ncsrc"]
COPY packages/ncs-6.6-cisco* /root/nso-6.6/packages/neds/
```

#### 4.2.3 COPY 명령어 수정

Dockerfile에서 `${pwd}` 변수는 지원되지 않습니다. 상대 경로를 직접 사용합니다.

**잘못된 방법 (도커 빌드 오류 발생):**
```dockerfile
COPY ${pwd}/nso-6.6.linux.x86_64.installer.bin .
```

**올바른 방법:**
```dockerfile
COPY nso-6.6.linux.x86_64.installer.bin .
```

### 4.3 현재 Dockerfile 전체 내용

```dockerfile
# Base Linux (Ubuntu 20.04) is used for the Cisco NSO local install
FROM ubuntu:20.04

USER root

# Metadata for this image
LABEL maintainer="Muhammad Rafi" \
      image.authors="murafi@cisco.com" \
      image.version="6.6"

# Set environment variable for the zero interaction
ARG DEBIAN_FRONTEND=noninteractive

# Update and install required packages to build Cisco NSO
RUN apt-get update && apt-get install -y \
    openssh-client \
    openssl \
    libexpat1 \
    libxml2-utils \
    libxml2 \
    xsltproc \
    default-jdk \
    build-essential \
    ant \
    net-tools

# Install Python3 and PIP (optional)
RUN apt-get update \
    && apt-get install -y python3-pip python3-dev \
    && cd /usr/local/bin \
    && ln -s /usr/bin/python3 python \
    && pip3 install --upgrade pip

# Working directory in use to perform the further actions
WORKDIR /root

# Copy NSO installer.bin file from the current directory to the /root directory
COPY nso-6.6.linux.x86_64.installer.bin .

# Generate SSH Key pair
RUN ssh-keygen -t rsa -C "$HOSTNAME" -f "$HOME/.ssh/id_rsa" -P "" && cat ~/.ssh/id_rsa.pub

# Install Cisco NSO as local install
RUN sh nso-6.6.linux.x86_64.installer.bin --local-install ~/nso-6.6

# Delete the nso installer after the container is built
RUN rm -f nso-6.6.linux.x86_64.installer.bin

# Update ~/.bashrc with 'source $HOME/nso-6.6/ncsrc'
RUN echo "source $HOME/nso-6.6/ncsrc" >> ~/.bashrc

# Set the shell to use bash and source ncsrc
SHELL ["/bin/bash", "-c", "source $HOME/nso-6.6/ncsrc"]

# Copy NSO packages to the container package directory
COPY packages/ncs-6.6-cisco* /root/nso-6.6/packages/neds/

# Expose port required for Cisco NSO Server
EXPOSE 2022 2024 8080 8888
```

## 5. Docker 이미지 빌드 프로세스

### 5.1 빌드 전 점검

빌드를 진행하기 전에 다음을 확인합니다.

```bash
# 현재 디렉토리 확인
pwd

# 필수 파일 존재 확인
[ -f nso-6.6.linux.x86_64.installer.bin ] && echo "NSO 설치 파일: OK" || echo "NSO 설치 파일: 없음"
[ -f packages/ncs-6.6-cisco-ios-6.110.3.tar.gz ] && echo "IOS NED: OK" || echo "IOS NED: 없음"
[ -f packages/ncs-6.6-cisco-nx-5.31.tar.gz ] && echo "NX NED: OK" || echo "NX NED: 없음"
[ -f Dockerfile ] && echo "Dockerfile: OK" || echo "Dockerfile: 없음"
```

### 5.2 이미지 빌드

**기본 빌드 명령:**

```bash
docker build -t cisco-nso-dev:6.6 .
```

**상세 옵션을 포함한 빌드:**

```bash
# 빌드 진행 상황을 상세히 표시하며 빌드
docker build -t cisco-nso-dev:6.6 . \
  --progress=plain \
  --no-cache

# 빌드 로그를 파일에 저장하면서 빌드
docker build -t cisco-nso-dev:6.6 . \
  --progress=plain 2>&1 | tee build.log
```

### 5.3 빌드 진행 과정

빌드가 시작되면 다음과 같은 단계를 거칩니다.

| 단계 | 예상 시간 | 작업 내용 |
|------|---------|---------|
| 1. Base 이미지 다운로드 | ~30초 | Ubuntu 20.04 이미지 풀 |
| 2. 패키지 업데이트 | ~1분 | apt-get update 실행 |
| 3. 패키지 설치 | ~2분 | 필수 패키지 설치 (JDK, Python, SSH 등) |
| 4. NSO 설치 | ~3-5분 | NSO 6.6 로컬 설치 |
| 5. 이미지 최적화 | ~1분 | 레이어 컴프레션 및 최적화 |

**전체 빌드 시간: 약 10-15분**

### 5.4 빌드 완료 확인

```bash
# 빌드 완료 후 이미지 목록 확인
docker images | grep cisco-nso-dev
```

**예상 출력:**

```
REPOSITORY          TAG    IMAGE ID       CREATED         SIZE
cisco-nso-dev       6.6    xxxxxxxxxxxxx  X minutes ago   1.5GB
```

### 5.5 빌드 실패 시 대응

```bash
# 이전 빌드 캐시 제거 후 재빌드
docker build -t cisco-nso-dev:6.6 . --no-cache

# 빌드 중 특정 단계에서 실패한 경우, 해당 레이어까지 진행한 임시 이미지 실행
docker build -t cisco-nso-dev:6.6 . --progress=plain 2>&1 | grep "Step"
```

## 6. Docker 컨테이너 배포

### 6.1 컨테이너 실행

빌드 완료 후 컨테이너를 실행합니다.

```bash
# 기본 컨테이너 실행
docker run -itd \
  --hostname nso \
  --name cisco-nso-dev \
  -p 2024:2024 \
  -p 8080:8080 \
  cisco-nso-dev:6.6
```

**옵션 설명:**

| 옵션 | 설명 |
|------|------|
| `-itd` | 대화형(-i), 의사 터미널(-t), 백그라운드 실행(-d) |
| `--hostname nso` | 컨테이너 호스트명 설정 |
| `--name cisco-nso-dev` | 컨테이너 이름 |
| `-p 2024:2024` | 호스트 2024 포트를 컨테이너 2024 포트에 매핑 (SSH CLI) |
| `-p 8080:8080` | 호스트 8080 포트를 컨테이너 8080 포트에 매핑 (Web UI) |

### 6.2 포트 매핑 설정

NSO는 다음 포트들을 사용합니다. 필요에 따라 추가로 매핑할 수 있습니다.

```bash
# 모든 포트를 매핑하여 실행
docker run -itd \
  --hostname nso \
  --name cisco-nso-dev \
  -p 2022:2022 \
  -p 2024:2024 \
  -p 8080:8080 \
  -p 8888:8888 \
  cisco-nso-dev:6.6
```


```bash

docker run -itd `
  --hostname nso `
  --name cisco-nso-dev `
  -p 2022:2022 `
  -p 2024:2024 `
  -p 8080:8080 `
  -p 8888:8888 `
  cisco-nso-dev:6.6
  
```

| 포트 | 프로토콜 | 용도 | 설명 |
|------|--------|------|------|
| 2022 | SSH | NETCONF over SSH | 네트워크 장비 관리 |
| 2024 | SSH | NSO CLI | 명령어 인터페이스 |
| 8080 | HTTP | Web UI | 웹 기반 관리 인터페이스 |
| 8888 | RESTCONF | REST API | 프로그래매틱 인터페이스 |

### 6.3 컨테이너 상태 확인

```bash

docker ps | grep cisco-nso-dev
```

```
CONTAINER ID    IMAGE               COMMAND   CREATED          STATUS              PORTS
xxxxxxxxxxxxx   cisco-nso-dev:6.6   "bash"    2 minutes ago    Up 2 minutes        0.0.0.0:2024->2024/tcp, 0.0.0.0:8080->8080/tcp   cisco-nso-dev
```


## 7. NSO 인스턴스 초기화 및 구성

### 7.1 NSO 환경 접속 및 설정

컨테이너가 실행된 후 NSO 환경을 초기화합니다.

```bash
# 컨테이너 bash 접속 (상호작용 가능한 터미널)
docker exec -it cisco-nso-dev bash

# 컨테이너 내부에서 현재 디렉토리 확인
root@nso:/root# pwd
/root
```

**설명**: Docker 컨테이너 내부에서 작업할 때는 홈 디렉토리(`/root`)가 자동으로 설정되어 있습니다. 모든 경로는 절대 경로(`/root/...`)를 사용하는 것이 안정적입니다.

### 7.2 NED 패키지 추출

컨테이너에 복사된 NED 패키지를 압축 해제합니다.

**호스트 PC 터미널에서:**

```bash
# 컨테이너 내 NED 패키지 파일 확인
docker exec cisco-nso-dev bash -c "ls -lh /root/nso-6.6/packages/neds/"
```

**예상 출력:**

```
-rw-r--r-- root root 69255236 Oct 30 09:56 ncs-6.6-cisco-ios-6.110.3.tar.gz
-rw-r--r-- root root 8453190 Oct 28 12:56 ncs-6.6-cisco-nx-5.31.tar.gz
```

**NED 패키지 압축 해제:**

```bash
docker exec cisco-nso-dev bash -c "cd /root/nso-6.6/packages/neds && tar -xzf ncs-6.6-cisco-ios-6.110.3.tar.gz && tar -xzf ncs-6.6-cisco-nx-5.31.tar.gz && rm *.tar.gz"
```

**또는 개별적으로 실행:**

```bash
# 1) 컨테이너 bash 진입
docker exec -it cisco-nso-dev bash

# 2) 컨테이너 내부에서 다음 명령어 실행
cd /root/nso-6.6/packages/neds
tar -xzf ncs-6.6-cisco-ios-6.110.3.tar.gz
tar -xzf ncs-6.6-cisco-nx-5.31.tar.gz

# 압축 해제 결과 확인
ls -la | grep cisco

# 압축 파일 정리 (공간 절약)
rm *.tar.gz
```

**압축 해제 확인:**

```bash
docker exec cisco-nso-dev bash -c "ls -la /root/nso-6.6/packages/neds/ | grep cisco"
```

**예상 출력:**

### 7.3 NSO 인스턴스 생성

ncs-setup 유틸리티를 사용하여 NSO 인스턴스를 생성합니다.

**호스트 PC 터미널에서:**

```bash
# 컨테이너에서 NSO 버전 확인
docker exec cisco-nso-dev bash -c "source /root/nso-6.6/ncsrc && ncs --version"
```

**예상 출력:**

```
6.6
```

**NSO 인스턴스 생성 (컨테이너 내부에서):**

```bash
# 1) 컨테이너 bash 진입
docker exec -it cisco-nso-dev bash

# 2) 컨테이너 내부에서 다음 명령어 실행
cd /root
source /root/nso-6.6/ncsrc

ncs-setup \
  --package /root/nso-6.6/packages/neds/cisco-ios-cli-6.110 \
  --package /root/nso-6.6/packages/neds/cisco-nx-cli-5.31 \
  --dest /root/ncs-instance
```

**또는 한 줄로 실행:**

```bash
docker exec cisco-nso-dev bash -c "source /root/nso-6.6/ncsrc && ncs-setup --package /root/nso-6.6/packages/neds/cisco-ios-cli-6.110 --package /root/nso-6.6/packages/neds/cisco-nx-cli-5.31 --dest /root/ncs-instance"
```

```

...
```

### 7.4 NSO 서비스 시작

**호스트 PC 터미널에서:**

```bash
# NSO 서비스를 컨테이너 백그라운드에서 시작
docker exec -d cisco-nso-dev bash -c "cd /root/ncs-instance && source /root/nso-6.6/ncsrc && ncs"
```

**또는 대화형으로 포그라운드에서 실행 (NSO 로그를 실시간으로 보려면):**

```bash
# 새로운 터미널에서 실행
docker exec -it cisco-nso-dev bash -c "cd /root/ncs-instance && source /root/nso-6.6/ncsrc && ncs"
```

### 7.5 패키지 로드 확인

```bash
# NSO CLI 진입 (호스트 PC의 터미널에서)
docker exec -it cisco-nso-dev bash -c "cd /root/ncs-instance && source /root/nso-6.6/ncsrc && ncs_cli -C -u admin"
```

**설명**: 
- `cd /root/ncs-instance`: 절대 경로로 NCS 인스턴스 디렉토리로 이동
- `source /root/nso-6.6/ncsrc`: 절대 경로로 NSO 환경 변수 로드
- `ncs_cli -C -u admin`: C-style CLI로 admin 계정으로 접속

**컨테이너 내부에서 접속하는 경우** (이미 컨테이너에 진입한 상태):
```bash
# 컨테이너 내부에서
cd /root/ncs-instance
source /root/nso-6.6/ncsrc
ncs_cli -C -u admin

# NSO CLI 내에서 패키지 확인
admin@ncs> show packages package cisco* | display-level 1 | nomore
```

**예상 출력:**

```
packages package cisco-ios-cli-6.110
  package-version 6.110.3
  description "NED package for the Cisco IOS"
  oper-status up
packages package cisco-nx-cli-5.31
  package-version 5.31
  description "NED package for the Cisco Nexus"
  oper-status up
```

## 8. Docker Hub 이미지 활용

### 8.1 Docker Hub에서 이미지 다운로드

로컬에서 빌드하지 않고 Docker Hub에서 미리 빌드된 이미지를 사용할 수 있습니다.

```bash
# Docker Hub에서 이미지 다운로드
docker pull homesever/cisco-nso-dev:6.6

# 다운로드 완료 확인
docker images | grep homesever
```

### 8.2 Docker Hub 이미지로 컨테이너 실행

```bash
# Docker Hub 이미지로 컨테이너 실행
docker run -itd \
  --hostname nso \
  --name cisco-nso-dev \
  -p 2024:2024 \
  -p 8080:8080 \
  homesever/cisco-nso-dev:6.6
```

이후 절차는 섹션 7과 동일합니다.
